      Manual

      Scenarios is the table name of all content. That includes APs, modules, both SFS and PFS, etc. The following are all the fields in the table and its values.

      Field Values Required Notes
      id Number Yes Autofilled by tracker.
      name String Yes For scenarios just fill in the name `Quest for Something`. For multiparts (scenarios or modules) fill out like this: `Something Quest, Part II: Something Returns`. For adventure paths preface the text with the AP name like this: `Rise of Something: #1 Burnt Something`. Finally preface any specials with `Special`, like: `Special: Year of the Something Lodge`.
      description Text Yes Copy paste the flavour text from paizo.com. Do not add any level range or non-story info like 'This scenario adds to the goals of the Dark Archive faction.' 90% of the time this is just the first paragraph.
      type String Yes Options: scenario, mod, ap, other
      season Number Yes Season number. Use `99` if this is not part of a season (APs, modules, etc.)
      number Number No Number of the scenario. If this is not a scenario, leave it empty.
      tier String No Human readable tier text. `1-5` or `8-24`, etc.
      levelrange String Yes By default the value is `01|02|03|04|05|06|etc...` Remove the levels (and pipes) that this is not appropriate for. The system uses this to filter. Example: `02|03|04|05` -> 2-5
      link String No Store link, needs to be a weblink.
      evergreen Boolean Yes Defaults to `0`, `1` means it is an evergreen.
      quest Boolean Yes Defaults to `0`, `1` means it is a quest.
      multitable Boolean Yes Defaults to `0`, `1` means it is multitable special.
      game OptionList Yes Defaults to `pfs`. Set to `sfs` if the content is Starfinder.
      archived_at Datetime No This needs to be set if content is retired. Only Simon can do this in the database, doesn't happen enough to code in this interface.
      created_at DateTime Yes Autofilled by tracker
      updated_at DateTime Yes Autofilled by tracker
      deleted_at DateTime Yes Autofilled by tracker. If this is set then the content is officially 'deleted'. This is effectively a soft delete.

      Adding Authors to Scenarios

      Authors are their own seperate entity in the database and are effectively just a name. You can connect 0 or more authors to a Scenario. You can also create Authors (soon TM). Please make sure a Author does not already exist, the code does not check for uniqueness.
